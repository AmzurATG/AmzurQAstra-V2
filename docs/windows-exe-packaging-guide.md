# QAstra — Windows EXE Packaging Guide
### React + Vite + FastAPI + PostgreSQL → Single `.exe`

---

## Overview

This guide explains how to package **QAstra** (React/Vite frontend + FastAPI backend + PostgreSQL) into a single Windows `.exe` file. When the user runs the exe:

1. Alembic migrations run automatically against the configured database
2. FastAPI starts silently in the background (no terminal window)
3. The browser opens automatically to `http://localhost:8000`
4. All configuration is read from a `.env` file the user sets up once

---

## Architecture

```
QAstra.exe  (PyInstaller bundled launcher)
├── Reads .env for DB + LLM + security credentials
├── Runs Alembic migrations (alembic upgrade head)
├── Starts FastAPI backend silently (port 8000)
├── FastAPI serves React build as static files (port 8000)
├── FastAPI serves /api/v1/* routes (REST API)
├── FastAPI serves /screenshots/* (browser-use screenshots)
└── Opens http://localhost:8000 in default browser
```

> **Single port design:** React is built as static files and served directly by FastAPI — no separate Vite dev server. Only port `8000` runs.

## Step 1 — The `.env` File (User Configures This)

The `.env` file sits **next to the `.exe`** in the installation folder. The user fills it in once before running the app.

```env
# .env — place this in the same folder as QAstra.exe

# ── Database (required) ───────────────────────────────────────
DATABASE_URL=postgresql://username:password@localhost:5432/qastra

# ── Security (required) ──────────────────────────────────────
SECRET_KEY=replace-with-a-random-string
ENCRYPTION_KEY=replace-with-a-32-byte-base64-key

# ── LLM (at least one provider required) ─────────────────────
DEFAULT_LLM_PROVIDER=litellm
LITELLM_API_KEY=your-litellm-key
LITELLM_API_BASE=https://your-litellm-proxy.example.com
LITELLM_MODEL=gpt-4o

# Or use direct provider keys:
# OPENAI_API_KEY=sk-...
# ANTHROPIC_API_KEY=sk-ant-...

# ── App Settings (optional) ──────────────────────────────────
ENVIRONMENT=production
DEBUG=false
LOG_LEVEL=INFO

# ── Redis (optional — leave blank to disable) ────────────────
# REDIS_URL=redis://localhost:6379/0

# ── Integrations (optional) ──────────────────────────────────
# SLACK_WEBHOOK_URL=https://hooks.slack.com/services/...

# ── Admin Account ────────────────────────────────────────────
ADMIN_EMAIL=admin@qastra.dev
ADMIN_PASSWORD=change-me-on-first-login
```

> ⚠️ The `.env` file is **never bundled inside the exe**. It lives alongside it so users can edit it anytime without rebuilding.


## Step 3 — Backend Changes for EXE Mode

### 3a. `backend/config.py` — Frozen-Aware `.env` Resolution

The current `config.py` resolves `.env` relative to `__file__` (the backend directory). Inside a PyInstaller bundle, `__file__` points to the temporary `sys._MEIPASS` extraction folder — the user's `.env` won't be found there.

**Change the `_BACKEND_DIR` and `env_file` resolution:**

```python
"""
Application settings loaded from environment / .env.
"""
from __future__ import annotations

import sys
import json
from pathlib import Path
from typing import Any, List, Optional

from pydantic import AliasChoices, Field, field_validator
from pydantic_settings import BaseSettings, SettingsConfigDict

# In frozen exe: __file__ is inside sys._MEIPASS (temp extraction dir).
# _BACKEND_DIR is used for code-relative paths (alembic, static files).
_BACKEND_DIR = Path(__file__).resolve().parent

def _resolve_env_file() -> str:
    """
    Resolve .env location:
    - Frozen exe: .env sits next to QAstra.exe (sys.executable's parent)
    - Dev mode:   .env sits inside backend/
    """
    if getattr(sys, 'frozen', False):
        return str(Path(sys.executable).parent / ".env")
    return str(_BACKEND_DIR / ".env")

def _resolve_app_root() -> Path:
    """
    Resolve the application root directory for runtime data (logs, screenshots, storage).
    - Frozen exe: the folder containing QAstra.exe
    - Dev mode:   the project root (parent of backend/)
    """
    if getattr(sys, 'frozen', False):
        return Path(sys.executable).parent
    return _BACKEND_DIR.parent


_APP_ROOT = _resolve_app_root()


class Settings(BaseSettings):
    model_config = SettingsConfigDict(
        env_file=_resolve_env_file(),
        env_file_encoding="utf-8",
        extra="ignore",
    )

    # ... (all existing fields remain unchanged) ...

    # Override paths that must resolve relative to the exe (not _MEIPASS)
    SCREENSHOTS_DIR: str = str(_APP_ROOT / "screenshots")
    LOG_DIR: str = str(_APP_ROOT / "logs")
    STORAGE_LOCAL_PATH: str = str(_APP_ROOT / "storage")
    UPLOAD_DIR: str = str(_APP_ROOT / "uploads")

settings = Settings()
```

**Key change:** `_resolve_env_file()` returns the `.env` path next to `sys.executable` when frozen, and `_resolve_app_root()` ensures logs/screenshots/storage are created next to the exe — not inside the temp `_MEIPASS` folder.

### 3b. `backend/main.py` — Serve React Build + Catch-All Route

Add static file serving and a catch-all route for React Router. These must be registered **after** the API router and screenshot mount:

```python
# Add these imports at the top of main.py
from fastapi.responses import FileResponse

def create_application() -> FastAPI:
    # ... existing app setup, middleware, API router, screenshots mount ...

    # Register global exception handlers
    register_exception_handlers(app)

    # ── Serve React static build (production / exe mode) ────────────
    static_dir = Path(settings.STATIC_DIR) if hasattr(settings, 'STATIC_DIR') \
        else Path(__file__).resolve().parent / "static"

    if static_dir.is_dir():
        # Serve /assets/* (JS, CSS, images produced by Vite)
        assets_dir = static_dir / "assets"
        if assets_dir.is_dir():
            app.mount("/assets", StaticFiles(directory=str(assets_dir)), name="frontend-assets")

        # Catch-all: serve index.html for any non-API, non-static route (React Router)
        @app.get("/{full_path:path}")
        async def serve_frontend(full_path: str):
            return FileResponse(str(static_dir / "index.html"))

    return app
```

> **Order matters:** The catch-all route must be defined **after** all API routers and mounts. Otherwise it would swallow `/api/v1/*` and `/screenshots/*` requests.

### 3c. Alembic — Database Migrations in EXE Context

The launcher runs `alembic upgrade head` before starting the server. For this to work inside the bundled exe, Alembic needs to find its config and migration scripts inside `_MEIPASS/backend/`.

No changes needed to `alembic/env.py` — it already resolves paths via `sys.path` manipulation. The launcher handles `cwd` correctly (see Step 4).

## Step 4 — The Silent Launcher

This is the entry point compiled into `QAstra.exe`. It:
1. Runs Alembic migrations against the user's database
2. Starts FastAPI as a hidden background process
3. Waits until the server is ready
4. Opens the browser automatically

```python
# launcher.py  (project root)
import subprocess
import sys
import os
import time
import webbrowser
import threading
import urllib.request


def get_base_path():
    """Code/data root: _MEIPASS when frozen, project root in dev."""
    if getattr(sys, 'frozen', False):
        return sys._MEIPASS
    return os.path.dirname(__file__)


def get_exe_dir():
    """Directory where the exe lives (where .env also lives)."""
    if getattr(sys, 'frozen', False):
        return os.path.dirname(sys.executable)
    return os.path.dirname(__file__)


def wait_for_server(url, timeout=60):
    """Poll the server until it responds or times out."""
    start = time.time()
    while time.time() - start < timeout:
        try:
            urllib.request.urlopen(url, timeout=2)
            return True
        except Exception:
            time.sleep(0.5)
    return False


def run_migrations(backend_dir, env):
    """Run Alembic migrations before starting the server."""
    try:
        result = subprocess.run(
            [sys.executable, "-m", "alembic", "upgrade", "head"],
            cwd=backend_dir,
            env=env,
            capture_output=True,
            text=True,
            timeout=120,
            creationflags=subprocess.CREATE_NO_WINDOW,
        )
        if result.returncode != 0:
            # Log migration failure — write to a file next to the exe
            err_path = os.path.join(get_exe_dir(), "migration_error.log")
            with open(err_path, "w") as f:
                f.write(f"STDOUT:\n{result.stdout}\n\nSTDERR:\n{result.stderr}\n")
            return False
        return True
    except Exception as e:
        err_path = os.path.join(get_exe_dir(), "migration_error.log")
        with open(err_path, "w") as f:
            f.write(f"Migration exception: {e}\n")
        return False


def start_backend(backend_dir, env):
    """Start uvicorn as a hidden subprocess."""
    proc = subprocess.Popen(
        [sys.executable, "-m", "uvicorn", "main:app",
         "--host", "127.0.0.1", "--port", "8000"],
        cwd=backend_dir,
        env=env,
        stdout=subprocess.DEVNULL,
        stderr=subprocess.DEVNULL,
        creationflags=subprocess.CREATE_NO_WINDOW,
    )
    return proc


if __name__ == "__main__":
    base        = get_base_path()
    backend_dir = os.path.join(base, "backend")
    exe_dir     = get_exe_dir()

    # Build environment — tell config.py where .env lives
    env = os.environ.copy()
    env["ENV_FILE_PATH"] = os.path.join(exe_dir, ".env")

    # 1. Run database migrations
    if not run_migrations(backend_dir, env):
        import ctypes
        ctypes.windll.user32.MessageBoxW(
            0,
            "Database migration failed.\n\n"
            "Check migration_error.log next to QAstra.exe for details.\n"
            "Verify your DATABASE_URL in .env is correct and the database is running.",
            "QAstra — Startup Error",
            0x10,  # MB_ICONERROR
        )
        sys.exit(1)

    # 2. Start FastAPI server
    proc = start_backend(backend_dir, env)
    url  = "http://127.0.0.1:8000"

    # 3. Open browser once server is ready
    def open_browser():
        if wait_for_server(f"{url}/health"):
            webbrowser.open(url)

    t = threading.Thread(target=open_browser, daemon=True)
    t.start()

    try:
        proc.wait()
    except KeyboardInterrupt:
        proc.terminate()
```

## Playwright Bundling (browser-use)

QAstra's functional testing feature uses `browser-use` which depends on Playwright + Chromium. Two options:

### Option A — User installs Chromium separately (simpler)

Include in `README.txt`:
```
Before first use, open a terminal and run:
  playwright install chromium
```

### Option B — Bundle Chromium in the exe (larger but zero-setup)

1. Find the Chromium path: `python -c "from playwright._impl._driver import compute_driver_executable; print(compute_driver_executable())"`
2. Add it to `datas` in `qastra.spec`:
   ```python
   datas=[
       ('backend', 'backend'),
       # Adjust the path to match your Playwright install location
       (r'C:\Users\<YOU>\AppData\Local\ms-playwright\chromium-*', 'ms-playwright/chromium'),
   ],
   ```

> Bundling Chromium adds ~150–200 MB to the exe size.

## Key Design Decisions

| Decision | Choice | Reason |
|---|---|---|
| Frontend serving | FastAPI serves React build at `/` | Single port, no Vite dev server needed in production |
| DB credentials | `.env` next to the exe | User-editable, not locked in the binary |
| DB migrations | `alembic upgrade head` on each launch | Ensures schema is always up-to-date |
| Terminal window | Hidden (`CREATE_NO_WINDOW` + `console=False`) | Clean UX, no scary black window |
| Browser launch | Polls `/health` endpoint before opening | Avoids blank page race condition |
| DB driver | `psycopg2-binary` (sync) + `asyncpg` (async) | No PostgreSQL client install required; async for FastAPI |
| `.env` location | `sys.executable` parent when frozen | Works correctly in both dev and exe contexts |
| Runtime data paths | Resolved via `_resolve_app_root()` | Logs, screenshots, storage go next to exe — not inside temp `_MEIPASS` |
| Async event loop | `WindowsProactorEventLoopPolicy` | Required for `browser-use` subprocess support on Windows |

## Common Pitfalls & Fixes

| Problem | Fix |
|---|---|
| `ModuleNotFoundError` at runtime | Add missing module to `hiddenimports` in `.spec` |
| `.env` not found | Ensure it's next to `.exe`, not inside it |
| Database migration fails | Check `migration_error.log` next to the exe; verify `DATABASE_URL` |
| Browser opens blank page | Increase `timeout` in `wait_for_server()` (default 60s) |
| App crashes silently | Temporarily set `console=True` in spec to see errors |
| Static files 404 | Confirm `npm run build` outputs to `backend/static/` |
| `psycopg2` DLL errors | Ensure `psycopg2-binary` is in requirements, not `psycopg2` |
| Playwright "browser not found" | Run `playwright install chromium` or bundle Chromium (see above) |
| Screenshots/logs missing | Check that `_resolve_app_root()` points next to the exe |
| Alembic can't find migrations | Verify `('backend', 'backend')` is in spec `datas` |
| `asyncio` errors on Windows | Ensure `WindowsProactorEventLoopPolicy` is set in `main.py` |

## Dev vs Production Behavior

| Scenario | `.env` loaded from | Static files | Migrations | Logs/Screenshots |
|---|---|---|---|---|
| `uvicorn main:app --reload` (dev) | `backend/.env` | Not served (Vite proxy) | Run manually: `alembic upgrade head` | `../logs/`, `../screenshots/` |
| `python launcher.py` (dev) | Project root `.env` | `backend/static/` | Auto on launch | `./logs/`, `./screenshots/` |
| `QAstra.exe` (production) | Folder next to exe | `_MEIPASS/backend/static/` | Auto on launch | Next to exe |
